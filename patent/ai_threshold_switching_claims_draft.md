# 权利要求书草案

发明名称：一种基于谱特征学习的多用户混合预编码方案切换阈值预测方法

1. 一种基于谱特征学习的多用户混合预编码方案切换阈值预测方法，其特征在于，应用于预设固定系统配置下的多用户下行混合预编码通信系统中在至少两种候选数字预编码方案之间进行切换的场景，所述候选数字预编码方案至少包括奇异值分解 SVD 数字预编码方案和统一信道分解 UCD 数字预编码方案，所述方法包括：
获取与所述预设固定系统配置对应的多用户信道矩阵；
基于所述多用户信道矩阵构建共享模拟预编码矩阵和用户级块对角化降维信道；
从各用户的降维信道中提取用于表征用户间和数据流间信道谱分布差异的谱特征 token 集合；
根据多个信噪比采样点上 UCD 方案性能与 SVD 方案性能之间的差值曲线，确定当前信道样本对应的方案切换阈值标签；
利用所述谱特征 token 集合以及切换阈值标签训练阈值预测模型；
在在线推理阶段，将待判决信道样本的谱特征 token 集合输入所述阈值预测模型，得到预测切换阈值；
将当前工作信噪比与所述预测切换阈值比较，当当前工作信噪比不小于所述预测切换阈值时选用 UCD 数字预编码方案，否则选用 SVD 数字预编码方案。

2. 根据权利要求1所述的方法，其特征在于，所述基于所述多用户信道矩阵构建共享模拟预编码矩阵和用户级块对角化降维信道，包括：
根据各用户信道矩阵构建共享模拟预编码矩阵 F_RF；
对第 k 个用户，计算等效信道矩阵 G_k = H_k F_RF；
将除第 k 个用户之外其余用户的等效信道矩阵按行堆叠，得到第 k 个用户对应的干扰矩阵 G_k^{I}；
求取所述干扰矩阵 G_k^{I} 的零空间基 N_k；
构造第 k 个用户的降维信道 Ĝ_k = G_k N_k。

3. 根据权利要求1所述的方法，其特征在于，所述谱特征 token 集合通过如下方式获得：
对每个用户的降维信道 Ĝ_k 进行奇异值分解，

Ĝ_k = Ũ_k Σ̃_k Ṽ_k^H；

其中，Σ̃_k = diag(σ_{k,1}, σ_{k,2}, …, σ_{k,d})；
将右奇异向量矩阵 Ṽ_k 的前 d 列作为第 k 个用户的 SVD 局部数字预编码矩阵 F_k^SVD；
针对每个用户的每个数据流，构造对应 token，所述 token 至少包括奇异值对数项 log(σ_{k,i})、中心化对数奇异值项 log(σ_{k,i}) - (1/d)∑_{m=1}^{d} log(σ_{k,m})、功率占比项 σ_{k,i}^2 / ∑_{m=1}^{d} σ_{k,m}^2、用户归一化位置特征和流索引归一化位置特征中的三种；
将全部用户、全部数据流对应的 token 按集合形式组合，得到当前信道样本的谱特征 token 集合。

4. 根据权利要求1所述的方法，其特征在于，所述 UCD 数字预编码方案通过如下方式构造：
根据降维信道 Ĝ_k 的奇异值序列 σ_{k,1}, σ_{k,2}, …, σ_{k,d} 以及与噪声水平相关的加载因子 α_k，确定各数据流的功率加载系数 p_{k,i}；
构造加载后的增广奇异值

λ_{k,i} = sqrt(p_{k,i} σ_{k,i}^2 + α_k)；

定义统一对角目标

λ̄_k = exp((1/d)∑_{i=1}^{d} ln λ_{k,i})；

构造功率加载对角矩阵 Λ_k = diag(p_{k,1}, p_{k,2}, …, p_{k,d})，并基于旋转矩阵 Ω_k 构造第 k 个用户的 UCD 局部数字预编码矩阵

F_k^UCD = Ṽ_k Λ_k^{1/2} Ω_k。

5. 根据权利要求1所述的方法，其特征在于，所述根据多个信噪比采样点上 UCD 方案性能与 SVD 方案性能之间的差值曲线，确定当前信道样本对应的方案切换阈值标签，包括：
在多个信噪比采样点上分别计算 SVD 方案性能值 R_SVD(γ_i) 和 UCD 方案性能值 R_UCD(γ_i)；
构造性能差值，如式（1）所示：

Δ(γ_i) = R_UCD(γ_i) - R_SVD(γ_i)    （1）

将 UCD 方案开始优于 SVD 方案的最早信噪比位置定义为切换起点；
当存在相邻采样点 γ_i 和 γ_(i+1) 满足 Δ(γ_i) ≤ 0 且 Δ(γ_(i+1)) ≥ 0 时，通过插值确定切换阈值，如式（2）所示：

γ_th = γ_i + (0 - Δ(γ_i)) (γ_(i+1) - γ_i) / (Δ(γ_(i+1)) - Δ(γ_i))    （2）

并将所述 γ_th 作为当前信道样本的切换阈值标签。

6. 根据权利要求5所述的方法，其特征在于，当所述性能差值在全部采样信噪比点上均不小于零时，将阈值标签设置为低于最小采样信噪比的预设保护值，以表示始终选择 UCD 方案；当所述性能差值在全部采样信噪比点上均不大于零时，将阈值标签设置为高于最大采样信噪比的预设保护值，以表示始终选择 SVD 方案。

7. 根据权利要求1所述的方法，其特征在于，所述阈值预测模型为一种针对预设固定系统配置的谱特征注意力网络，所述阈值预测模型包括：
token 投影模块，用于将谱特征 token 投影至隐空间；
至少一层自注意力编码模块，用于对谱特征 token 集合进行集合级特征交互；
池化读出模块，用于对经过自注意力编码后的 token 隐状态进行加权汇聚，得到全局表征；
阈值回归头，用于根据所述全局表征输出预测切换阈值。

8. 根据权利要求7所述的方法，其特征在于，所述阈值预测模型还包括符号辅助判别分支，所述符号辅助判别分支在若干选定信噪比采样点上输出 UCD 方案是否优于 SVD 方案的符号判决结果；训练时，将阈值回归损失与所述符号辅助判别损失进行联合优化，以增强预测阈值对切换方向变化的敏感性。

9. 根据权利要求8所述的方法，其特征在于，所述符号辅助判别分支的训练标签满足：

s(γ_i) = sign(max(Δ(γ_i), 0))，

其中 Δ(γ_i) = R_UCD(γ_i) - R_SVD(γ_i)。

10. 根据权利要求1所述的方法，其特征在于，所述多个信噪比采样点采用粗搜索与细搜索相结合的方式生成，具体包括：
先在较大步长的粗信噪比网格上计算所述性能差值曲线；
根据粗网格上的符号变化区间定位候选阈值所在区间；
再在候选阈值所在区间内按较小步长进行细粒度性能计算，以获得更精确的阈值标签。

11. 根据权利要求1所述的方法，其特征在于，所述方案性能值 R_SVD(γ_i) 和 R_UCD(γ_i) 至少包括和速率、误码率和比特交织编码调制广义互信息中的一种；并且在计算不同候选方案性能值时，对同一信道样本复用相同的调制符号样本和相同的噪声样本，以降低阈值标签生成过程中的随机偏差。

12. 根据权利要求1所述的方法，其特征在于，在线推理阶段的方案切换规则满足式（5）和式（6）：

选择 UCD， 当 γ_work ≥ γ_th^pred    （5）

选择 SVD， 当 γ_work < γ_th^pred    （6）

其中 γ_work 为当前工作信噪比，γ_th^pred 为阈值预测模型输出的预测切换阈值。

13. 一种基于谱特征学习的多用户混合预编码方案切换装置，其特征在于，包括：
信道获取模块，用于获取与预设固定系统配置对应的多用户信道矩阵；
降维信道构建模块，用于构建共享模拟预编码矩阵、等效信道矩阵和块对角化降维信道；
特征提取模块，用于提取谱特征 token 集合；
阈值预测模块，用于输出候选数字预编码方案之间的预测切换阈值；
方案选择模块，用于根据当前工作信噪比与所述预测切换阈值之间的比较结果，在 SVD 方案和 UCD 方案之间进行选择。

## 说明

这一版权利要求故意把核心保护点放在“AI 阈值切换”上，而不是把 `SVD`、`UCD` 本身当创新点。更具体地说，主张重点是：

- 用块对角化降维信道的谱特征而不是原始信道矩阵直接建模；
- 把“UCD 何时开始优于 SVD”定义成可学习的切换阈值标签；
- 在固定系统配置下对单一阈值模型进行训练；
- 通过阈值回归加符号辅助判别联合训练模型；
- 在线只预测一个阈值，再按信噪比执行低复杂度切换。

如果后面要继续打磨成正式版，最值得强化的是：

- 把“为什么选这些谱特征 token”写成技术效果；
- 把“阈值标签生成的粗细两级搜索”写成从属权利要求里的计算量优化点；
- 根据你最想保的核心，决定独立权利要求是否要把“符号辅助分支”提前到独立权利要求里。

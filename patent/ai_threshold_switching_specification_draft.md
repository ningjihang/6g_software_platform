# 说明书草案

发明名称：一种基于谱特征学习的多用户混合预编码方案切换阈值预测方法

## 技术领域

[0001] 本发明属于无线通信与智能信号处理技术领域，具体涉及一种面向多用户下行混合预编码系统的数字预编码方案切换方法，尤其涉及一种基于块对角化降维信道谱特征学习的 SVD/UCD 切换阈值预测方法。

## 背景技术

[0002] 多输入多输出通信系统能够充分利用空间自由度提升频谱效率，在大规模天线阵列、毫米波传输以及多用户下行场景中具有广泛应用价值。由于完全数字预编码需要为每根发射天线配置独立射频链路，系统硬件复杂度和功耗较高，因此工程上通常采用由模拟预编码与数字预编码共同组成的混合预编码结构。

[0003] 在多用户混合预编码系统中，数字预编码部分直接影响各用户的等效子信道结构和接收端性能。奇异值分解 SVD 方案具有结构清晰、实现简单、低中等信噪比区间性能稳定等特点；统一信道分解 UCD 方案则通过对子信道增益进行均衡化处理，在部分信道样本和较高信噪比区间能够取得比 SVD 更好的性能。

[0004] 然而，对固定系统配置下的不同信道样本而言，SVD 方案与 UCD 方案之间的优劣关系并不恒定。对于部分信道样本，SVD 方案在低信噪比区间性能较优，而当信噪比提高后，UCD 方案开始优于 SVD 方案。若始终固定采用单一数字预编码方案，则容易在部分信道条件下造成性能损失。

[0005] 现有方案中，通常通过经验阈值或对多个信噪比点进行穷举比较来决定方案切换。该类方法存在以下不足：一是需要反复计算不同数字预编码方案在多个信噪比点下的接收性能，在线复杂度较高；二是切换依据通常缺少对块对角化降维信道内部谱结构差异的精细表征；三是对于大量信道样本，阈值标注与在线切换效率较低。

[0006] 因此，需要一种能够在预设固定系统配置下，利用信道样本自身的降维谱结构信息，快速预测 SVD 方案与 UCD 方案之间切换阈值的方法，从而在降低在线计算复杂度的同时，提高方案切换准确性和系统整体性能。

## 发明内容

### 要解决的技术问题

[0007] 本发明旨在提供一种基于谱特征学习的多用户混合预编码方案切换阈值预测方法，以解决固定系统配置下 SVD 与 UCD 数字预编码方案之间切换依赖穷举比较、在线复杂度较高以及切换依据不够精确的问题。

### 技术方案

[0008] 为实现上述目的，本发明采用如下技术方案。

[0009] 设系统包括 K 个用户，基站发射天线数为 N_t，第 k 个用户的接收天线数为 N_r，系统射频链路数为 N_RF，每个用户的数据流数为 d。第 k 个用户的下行信道矩阵记为 H_k。发送端采用混合预编码结构，其中共享模拟预编码矩阵记为 F_RF，多用户数字预编码矩阵记为 F_BB，则第 k 个用户的接收信号可表示为

y_k = H_k F_RF F_BB s + n_k，    （1）

[0009a] 其中，s 为发送符号向量，n_k 为接收噪声向量。对于调制方式为 M 阶 QAM 的情形，设每个复符号携带 b = log_2(M) 个比特，则发送端首先将二进制比特序列映射为复星座符号，得到每个用户各数据流上的调制符号向量。

[0009b] 对于第 k 个用户的第 i 个数据流，发送符号可记为 x_{k,i}，其取值属于归一化 QAM 星座集合 X_M。将所有用户和所有数据流对应的发送符号按列组合后，得到式（1）中的发送符号向量 s。

[0010] 本发明首先基于多用户信道矩阵构建共享模拟预编码矩阵。对于第 k 个用户，对其信道矩阵 H_k 进行奇异值分解，可表示为

H_k = U_k Σ_k V_k^H。    （2）

[0011] 从右奇异向量矩阵 V_k 中提取主导列向量作为候选发射方向，并将不同用户的候选方向按列拼接得到相位源矩阵 C。对所述相位源矩阵仅保留相位信息并实施恒模归一化，构建共享模拟预编码矩阵

F_RF = (1 / sqrt(N_t)) exp(j·angle(C))。    （3）

[0012] 在共享模拟预编码矩阵 F_RF 已知时，第 k 个用户的等效信道矩阵表示为

G_k = H_k F_RF。    （4）

[0013] 为消除用户间干扰，对于第 k 个用户，将除第 k 个用户之外其余用户的等效信道矩阵按行堆叠，得到第 k 个用户对应的干扰矩阵

G_k^{I} = [G_1^T, …, G_{k-1}^T, G_{k+1}^T, …, G_K^T]^T。    （5）

[0014] 求取干扰矩阵 G_k^{I} 的零空间基 N_k，并将第 k 个用户的等效信道矩阵投影到该零空间内，得到块对角化降维信道

Ĝ_k = G_k N_k。    （6）

[0014a] 进一步地，将各用户的局部数字预编码矩阵与对应零空间基组合后，可构造多用户数字预编码矩阵

F_BB = [N_1 F_1, N_2 F_2, …, N_K F_K]。    （6a）

[0015] 对降维信道 Ĝ_k 进行奇异值分解，可表示为

Ĝ_k = Ũ_k Σ̃_k Ṽ_k^H，    （7）

其中，

Σ̃_k = diag(σ_{k,1}, σ_{k,2}, …, σ_{k,d})。    （8）

[0015a] 所述奇异值对角矩阵 Σ̃_k 可进一步示意为

Σ̃_k =
[ σ_{k,1}      0          …          0      ]
[    0      σ_{k,2}       …          0      ]
[    ⋮         ⋮         ⋱          ⋮      ]。    （8a）
[    0         0          …      σ_{k,d}   ]

[0016] 在 SVD 数字预编码方案中，直接选取右奇异向量矩阵 Ṽ_k 的前 d 列作为第 k 个用户的局部数字预编码矩阵，即

F_k^{SVD} = Ṽ_k(:,1:d)。    （9）

[0017] 相应地，可选取 Ũ_k 的前 d 列作为接收端线性处理矩阵，使第 k 个用户的降维信道近似分解为 d 个相互独立的并行子信道，各子信道增益由奇异值 σ_{k,1}, σ_{k,2}, …, σ_{k,d} 决定。

[0017a] 在 SVD 方案下，经接收端线性处理后，第 k 个用户的等效接收向量可表示为

z_k^{SVD} = Ũ_k^H Ĝ_k F_k^{SVD} x_k + n_k^{SVD} = Σ̃_k x_k + n_k^{SVD}，    （9a）

[0017b] 其中，发送符号向量 x_k 可写为

x_k = [x_{k,1}, x_{k,2}, …, x_{k,d}]^T。    （9b）

其中，x_k 为第 k 个用户发送符号向量，n_k^{SVD} 为等效噪声向量。由此可见，SVD 方案下的等效信道矩阵为对角矩阵，各数据流对应的子信道增益分别由 σ_{k,1}, σ_{k,2}, …, σ_{k,d} 给出。

[0018] 在 UCD 数字预编码方案中，本发明不是直接采用原始奇异值构造并行子信道，而是引入与噪声水平相关的加载因子 α_k 以及数据流功率加载系数 p_{k,i}，以构造更均衡的等效子信道结构。对于第 i 个数据流，可定义功率加载系数为

p_{k,i} = max(μ_k - α_k / σ_{k,i}^2, 0)，    （10）

其中，μ_k 为注水水位。

[0019] 在功率加载系数确定后，构造增广奇异值

λ_{k,i} = sqrt(p_{k,i} σ_{k,i}^2 + α_k)。    （11）

[0020] 为使不同数据流的等效子信道质量尽量均衡，定义统一对角目标

λ̄_k = exp((1/d)∑_{i=1}^{d} ln λ_{k,i})。    （12）

[0021] 令

Λ_k = diag(p_{k,1}, p_{k,2}, …, p_{k,d})，    （13）

[0021a] 所述功率加载对角矩阵 Λ_k 可进一步示意为

Λ_k =
[ p_{k,1}      0         …         0      ]
[    0      p_{k,2}      …         0      ]
[    ⋮         ⋮        ⋱         ⋮      ]。    （13a）
[    0         0         …      p_{k,d}   ]

并在右奇异向量矩阵 Ṽ_k 的基础上引入旋转矩阵 Ω_k，构造第 k 个用户的 UCD 局部数字预编码矩阵

F_k^{UCD} = Ṽ_k Λ_k^{1/2} Ω_k。    （14）

[0022] 在构造第 k 个用户的 UCD 局部数字预编码矩阵后，UCD 方案对应的等效信道可表示为

B_k^{UCD} = W_k^H Ĝ_k F_k^{UCD}。    （14a）

[0022a] 其中，B_k^{UCD} 为上三角等效信道矩阵，其对角元素近似满足

b_{k,1,1} ≈ b_{k,2,2} ≈ … ≈ b_{k,d,d} ≈ λ̄_k。    （14b）

[0022aa] 所述上三角等效信道矩阵 B_k^{UCD} 可进一步示意为

B_k^{UCD} =
[ b_{k,1,1}  b_{k,1,2}  …  b_{k,1,d} ]
[     0      b_{k,2,2}   …  b_{k,2,d} ]
[     ⋮         ⋮       ⋱      ⋮      ]。    （14bb）
[     0          0       …  b_{k,d,d} ]

[0022b] 因此，UCD 方案不再是简单对角结构，而是具有近似统一对角元素的上三角结构。与 SVD 方案相比，UCD 方案通过对子信道增益进行均衡化处理，有助于在部分信道样本和较高信噪比区间改善接收检测性能。

[0022c] 对于 UCD 方案，可将经过接收端均衡后的第 k 个用户等效接收信号表示为

z_k^{UCD} = B_k^{UCD} x_k + n_{k,eq}。    （14c）

其中，x_k 表示第 k 个用户发送符号向量，n_{k,eq} 表示均衡后的等效噪声项。

[0022d] 当采用上三角等效信道模型时，接收端可基于矩阵 B_k^{UCD} 对各数据流逐层或逐分量进行均衡与判决。

[0022e] 在一种实施方式中，可令

D_k = diag(b_{k,1,1}, b_{k,2,2}, …, b_{k,d,d})，    （14d）

并构造归一化上三角矩阵

\bar{B}_k = D_k^{-1} B_k^{UCD}。    （14e）

[0022f] 若记输入符号向量为

s_k = [s_{k,1}, s_{k,2}, …, s_{k,d}]^T，    （14f）

则 THP 型递推输出向量 u_k = [u_{k,1}, u_{k,2}, …, u_{k,d}]^T 的第 i 个分量可表示为

u_{k,i} = Mod(s_{k,i} - ∑_{j=i+1}^{d} \bar{b}_{k,i,j} u_{k,j})。    （14g）

[0022g] 经过局部数字预编码矩阵作用后，第 k 个用户的发射向量可表示为

\tilde{x}_k = F_k^{UCD} u_k。    （14h）

[0023] 在预设信噪比采样点 γ_i 上，分别计算 SVD 方案和 UCD 方案对应的性能值 R_SVD(γ_i) 和 R_UCD(γ_i)，并定义性能差值

Δ(γ_i) = R_UCD(γ_i) - R_SVD(γ_i)。    （15）

[0024] 当存在相邻采样点 γ_i 和 γ_(i+1) 使得 UCD 方案由不优于 SVD 方案变为优于 SVD 方案时，通过插值确定切换阈值 γ_th，即

γ_th = γ_i + (0 - Δ(γ_i))(γ_(i+1) - γ_i) / (Δ(γ_(i+1)) - Δ(γ_i))。    （16）

[0025] 将降维信道谱特征 token 集合与对应阈值 γ_th 组成训练样本，训练针对固定系统配置的谱特征注意力网络。所述谱特征 token 至少包括奇异值对数项、中心化对数奇异值项、功率占比项以及位置特征项，其中可写为

t_{k,i} = [log(σ_{k,i}), log(σ_{k,i}) - (1/d)∑_{m=1}^{d}log(σ_{k,m}), σ_{k,i}^2 / ∑_{m=1}^{d}σ_{k,m}^2, p_k, q_i]。    （17）

[0026] 其中，p_k 表示用户归一化位置特征，q_i 表示流索引归一化位置特征。由于系统配置固定，因此每个信道样本对应的 token 数量固定为 K×d。

[0027] 在在线阶段，将待判决信道样本对应的谱特征 token 集合输入训练完成的阈值预测模型，得到预测切换阈值 γ_th^pred；再根据当前工作信噪比 γ_work 与 γ_th^pred 的关系选择数字预编码方案，即

γ_work ≥ γ_th^pred 时选择 UCD 方案；    （18）

γ_work < γ_th^pred 时选择 SVD 方案。    （19）

[0028] 在一种可选实现方式中，训练阶段还引入符号辅助判别分支，在若干选定信噪比点上判断 UCD 方案是否优于 SVD 方案，其辅助标签可表示为

s(γ_i) = sign(max(Δ(γ_i), 0))。    （20）

[0029] 在一种可选实现方式中，为降低阈值标签生成复杂度，先在较大步长的粗信噪比网格上比较方案性能，再在候选切换区间内按较小步长细化搜索。

[0030] 在一种可选实现方式中，为减小 Monte Carlo 评价误差，在比较 SVD 方案和 UCD 方案性能时，对同一信道样本复用相同的调制符号样本和相同的噪声样本。

[0030a] 对于 SVD 方案的接收评价，可基于并行子信道模型对各数据流分别计算比特交织编码调制广义互信息或误码率；对于 UCD 方案的接收评价，可基于均衡后的上三角等效信道计算对应的接收广义互信息或误码率。

[0030b] 在一种实现方式中，接收端通过对候选星座点与接收符号之间的距离进行度量，计算每个比特位的对数似然比 LLR，并基于所述 LLR 进一步估计比特交织编码调制广义互信息。对于第 l 个比特位，可写为

LLR_l = log \left( sum_{x in X_l^1} exp(-|z - x|^2 / sigma_n^2) \right) - log \left( sum_{x in X_l^0} exp(-|z - x|^2 / sigma_n^2) \right)。    （21）

[0030c] 其中，X_l^1 和 X_l^0 分别表示第 l 个比特位取值为 1 和 0 的候选星座点集合，z 表示均衡后的接收符号，sigma_n^2 表示等效噪声功率。

### 有益效果

[0031] 与现有技术相比，本发明至少具有如下有益效果：

1. 通过建立完整的混合预编码、块对角化降维、SVD 构造、UCD 构造和阈值预测链条，使方案切换建立在明确的通信物理模型和矩阵分解基础上。
2. 利用块对角化降维信道的谱结构特征而非单一经验统计量预测切换阈值，能够更准确地反映不同信道样本上的方案优劣变化。
3. 通过将穷举式在线方案比较转化为离线训练加在线阈值预测，显著降低在线计算复杂度。
4. 在固定系统配置下训练专用阈值模型，可增强模型与目标场景之间的匹配性，提高阈值预测精度。
5. 通过粗细两级信噪比搜索和统一随机样本复用策略，可兼顾阈值定位精度、样本生成效率与方案比较公平性。

## 具体实施方式

[0033] 以下结合具体实施例对本发明作进一步说明。应当理解，以下实施例仅用于解释本发明，而不用于限定本发明的保护范围。

### 实施例一：系统配置与共享模拟预编码

[0034] 设系统固定配置为：用户数 K = 2，发射天线数 N_t = 16，每用户接收天线数 N_r = 4，射频链路数 N_RF = 8，每用户数据流数 d = 4，调制方式为 64QAM。上述参数仅为示例，本发明也可适用于其他固定系统配置。

[0035] 对每个信道样本，首先生成多用户下行信道矩阵 H_1, H_2, …, H_K。对于每个用户，对 H_k 按式（2）进行奇异值分解，提取右奇异向量的主导列向量作为候选发射方向；再将不同用户候选方向按列拼接并提取相位，依据式（3）构造共享模拟预编码矩阵 F_RF。

### 实施例二：块对角化降维与 SVD 构造

[0036] 对于第 k 个用户，根据式（4）计算其等效信道矩阵 G_k；根据式（5）构造干扰矩阵 G_k^{I}；求取 G_k^{I} 的零空间基 N_k 后，根据式（6）构造降维信道 Ĝ_k。

[0037] 对 Ĝ_k 按式（7）进行奇异值分解，得到奇异值序列 σ_{k,1}, σ_{k,2}, …, σ_{k,d} 及其对应左右奇异向量矩阵。依据式（9），将右奇异向量矩阵的前 d 列取出，得到 SVD 局部数字预编码矩阵 F_k^{SVD}。

[0038] 由此，SVD 方案对应的等效子信道被分解为若干并行独立子信道，每个子信道的有效增益由对应奇异值决定。

[0038a] 在本实施例中，SVD 方案的接收端评价基于并行子信道模型进行，即先对各数据流分别实施均衡，再依据候选星座点与接收符号之间的欧氏距离计算对应比特位的对数似然比和广义互信息。

### 实施例三：UCD 构造

[0039] 在同一降维信道 Ĝ_k 上，UCD 方案首先根据奇异值序列与噪声相关加载因子 α_k 构造功率加载系数 p_{k,i}，可按式（10）实施类注水分配。

[0040] 在得到功率加载系数后，根据式（11）构造增广奇异值 λ_{k,i}，再按式（12）计算统一对角目标 λ̄_k。之后构造功率加载对角矩阵 Λ_k，并引入旋转矩阵 Ω_k，依据式（14）生成 UCD 局部数字预编码矩阵 F_k^{UCD}，再根据式（14a）至式（14c）得到对应的上三角等效信道与均衡后接收模型。

[0040a] 在一种实施方式中，进一步根据式（14d）至式（14h）构造归一化上三角矩阵、THP 型递推输出向量和最终发射向量，以完成基于 UCD 上三角等效信道的发送端预处理与接收端均衡判决。

[0041] 与 SVD 方案直接保留奇异值差异不同，UCD 方案通过统一对角目标对子信道强弱进行均衡，使上三角等效信道的对角元素更加接近，从而在部分信道样本和高信噪比条件下提升检测与译码性能。

[0041a] 在本实施例中，UCD 方案的接收端首先根据等效上三角信道实施线性均衡，然后依据均衡后的接收符号计算误码率或广义互信息，从而得到性能值 R_UCD(γ_i)。

### 实施例四：阈值标签生成

[0042] 在预设信噪比搜索范围内，选取多个信噪比采样点 γ_1, γ_2, …, γ_L。对于每个采样点，分别计算 SVD 方案和 UCD 方案的性能值，并按式（15）得到性能差值曲线。

[0043] 当性能差值在全部采样点上均不小于零时，标记该信道样本始终选择 UCD 方案；当性能差值在全部采样点上均不大于零时，标记该信道样本始终选择 SVD 方案；当性能差值存在由负转正的区间时，依据式（16）插值计算切换阈值 γ_th。

[0044] 为减少标签生成的计算量，可先在较大步长的粗信噪比网格上定位候选切换区间，再在候选切换区间内以较小步长实施细化搜索。

### 实施例五：谱特征构造与模型训练

[0045] 对每个用户、每个数据流，根据式（17）构造谱特征 token，并将全部 token 组合成当前信道样本的输入集合。由于系统配置固定，输入 token 数量固定为 K×d。

[0046] 将谱特征 token 集合与阈值标签 γ_th 组成训练数据，训练谱特征注意力网络，使模型学习信道谱结构与切换阈值之间的映射关系。在可选实施方式中，可同时引入式（20）和式（21）所定义的符号辅助标签，以增强模型对切换方向变化的识别能力。

### 实施例六：在线方案切换

[0047] 在模型训练完成后，对于待判决信道样本，重复实施共享模拟预编码构造、块对角化降维和谱特征提取过程，并将谱特征 token 集合输入模型，得到预测切换阈值 γ_th^pred。

[0048] 根据式（18）和式（19），当 γ_work 不小于 γ_th^pred 时，选择 UCD 数字预编码方案；当 γ_work 小于 γ_th^pred 时，选择 SVD 数字预编码方案，从而完成针对固定系统配置的低复杂度在线方案切换。

[0048a] 当完成方案选择后，发送端按照所选方案分别构造多用户数字预编码矩阵 F_BB，并与共享模拟预编码矩阵 F_RF 共同形成最终混合预编码矩阵；随后完成 QAM 调制后的信号发送，接收端依据所选方案对应的均衡和解调过程完成符号判决及性能评价。
